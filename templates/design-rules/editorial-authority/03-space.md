# Space & Rhythm — Editorial Authority

## 핵심 원칙

여백은 비어있는 것이 아니라 구조다.
타이포그래피가 리듬을 만들고, 여백이 호흡을 만든다.
Editorial은 **여유롭게** 숨 쉰다. 빽빽하게 채우지 않는다.

## Base Unit

**8px** 기준. 황금비(1.618)를 의식하되, 8의 배수에 맞춘다.

```
8px   — 마이크로: 라벨-인풋 간격
16px  — 인풋 패딩, 관련 요소 간격
24px  — 카드 내부 텍스트 간 여백
32px  — 카드 패딩
48px  — 컴포넌트 간 간격
80px  — 섹션 간 간격
120px — 히어로 섹션 상하 패딩
```

## 밀도 설정: Airy (기본)

Editorial의 밀도는 **항상 Airy**다.

- 섹션 패딩: 80px 이상
- 카드 패딩: 32px
- 본문 단락 간격: 24px
- 컴포넌트가 화면을 가득 채우면 안 된다
- "비어 보인다"는 것은 올바른 상태다

## 그리드

```css
display: grid;
grid-template-columns: repeat(12, 1fr);
gap: 24px;
max-width: 1080px;
margin: 0 auto;
padding: 0 40px;
```

본문 컨텐츠 영역은 12컬럼 중 **중앙 8컬럼** 사용.
사이드바 레이아웃은 8+4 또는 9+3.

## 최대 너비

```
전체 레이아웃: max-width 1080px
본문 텍스트 컬럼: max-width 680px (가독성 최적)
히어로 타이틀: max-width 800px
인용문 블록: max-width 600px
```

**680px**은 Editorial의 핵심 숫자다. 본문이 이보다 넓으면 안 된다.

## 정렬 원칙

- 텍스트: 항상 왼쪽 정렬 (중앙 정렬은 히어로 타이틀에만 허용)
- 이미지: 컨텐츠 영역 100% 너비 또는 그리드 컬럼에 맞춤
- 컴포넌트: 왼쪽 정렬 기본, 그리드에 snap
- 숫자: 오른쪽 정렬 (표에서)

## 수직 리듬

```
H1 위: 80px
H2 위: 64px
H3 위: 40px
본문 단락 간: 24px
리스트 항목 간: 8px
```

헤딩 위 여백은 아래 여백보다 항상 크다.
새로운 섹션이 시작된다는 신호를 여백으로 만든다.

## Claude에게

컴포넌트를 배치할 때 "너무 비어 보이지 않나?"라는 의심이 들면 그것이 올바른 상태다.
여백을 줄여서 해결하려 하지 마라. 콘텐츠가 부족한 것이 문제다.
