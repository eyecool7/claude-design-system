# Design Personality — Tech Precision

## 핵심 철학

> "데이터가 미학이다. 기능이 형태를 결정한다."

이 디자인은 **정보의 효율적 전달**이 목적이다.
모든 것은 그리드 위에 있고, 모든 색은 의미를 갖고, 모든 컴포넌트는 기능으로 정당화된다.
터미널과 대시보드의 미학. 코드의 정밀함을 UI로 옮긴다.

## 설계 원칙 (우선순위 순)

1. **Data is aesthetics** — 잘 정리된 데이터가 가장 아름다운 UI다
2. **Function determines form** — 기능이 없는 요소는 존재하지 않는다
3. **Everything on a grid** — 1px도 그리드를 벗어나지 않는다
4. **Monospace is identity** — 모노스페이스 타이포가 이 디자인의 정체성이다

## 이 Personality가 피하는 것

- 장식적 일러스트, 아이콘
- 둥근 모서리 (4px 초과)
- Soft shadow (glow만 허용)
- 세리프 폰트
- 중앙 정렬 (데이터 레이블 제외)
- 따뜻한 색 계열 (베이지, 크림, 테라코타)
- 부드러운 전환 (0.3s 이상)

## Claude에게

컴포넌트를 만들 때 "이것이 데이터를 더 잘 보여주는가?"를 물어라.
예쁘지만 정보가 없는 요소는 삭제한다.
모든 요소가 그리드 snap 되어 있는지 확인하라.
